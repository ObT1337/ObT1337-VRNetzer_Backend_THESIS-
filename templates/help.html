VRNetzer Web App HELP

The VrNetzer Web App is a flask webserver that acts both as a backend (the python part) and as a frontend (all the html and js files) in the form of user interfaces for the VR Module. It also has functionality that is meant to be used from a webbrowser, infact a webbrowser can be used in connection with a vr session.
Backend
- one per session
- serves the html pages for user interfaces (UI)
- multicast changes to the UI to all connected clients
- serves networkdata as textures
- holds the state of the session

Frontend
- client side (can run on a different machine than server)
- one for each player
- notify server of useractions (socketIO)
- communicates with VRModul via WebUI.js ("Middleman")

VRModule
- binary, can't be changed
- has fixed api
- in game webbrowser (Chromium)
- WebUI plugin for bi directional communication with Frontend
- downloads big textures directly from server
- config file tells it which server to connect to and wich html pages to load
